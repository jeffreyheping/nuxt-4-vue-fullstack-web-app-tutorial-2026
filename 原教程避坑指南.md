# 原教程避坑指南

> 教程来源：[Nuxt 4 Vue Fullstack Web App Tutorial 2026](https://www.noqta.tn/en/tutorials/nuxt-4-vue-fullstack-web-app-tutorial-2026)
>
> 本文档记录按照教程搭建 TaskFlow 项目时遇到的所有坑，以及修正方法。

---

## 核心发现：教程基于 @nuxt/ui v2，不要用 v3

教程写于 2026 年 3 月，执行 `pnpm add @nuxt/ui` 时默认装的是 **v2.x**（最新 2.22.3）。但在 2026 年 5 月搭建时，如果 `package.json` 中写了 `"@nuxt/ui": "^3.0.0"` 或不加版本号，npm/pnpm 会装 **v3.3.x**。

v2 和 v3 差异巨大，导致后续出现 5 个连环坑：

| 问题 | v2 行为 | v3 行为 |
|------|---------|---------|
| 表单组件 | `UFormGroup` ✅ | `UFormField`（改名） |
| Tailwind CSS | `@nuxtjs/tailwindcss` 自动管理 | 需手动创建 `main.css` |
| Google 字体 | 不依赖 `@nuxt/fonts` | 传递依赖自动加载，需显式禁用 |
| 根组件 | 无需 `UApp` | 必须 `<UApp>` 包裹 |
| 模态框表单 | 正常工作 | 灰色遮罩 bug |

**解决**：不用和 v3 死磕，降级到 `@nuxt/ui@^2.0.0`，完全复刻教程写法。

```json
// package.json
"@nuxt/ui": "^2.0.0",
```

## 坑 1：Prisma 版本未锁定

**现象**：执行 `pnpm dlx prisma generate` 时，下载到 Prisma 7.x，提示 `datasource.url` 字段已废弃。

**原因**：Prisma 7 有破坏性变更，教程基于 Prisma 6。

**修正**：
```json
"@prisma/client": "6.19.3",
"prisma": "6.19.3"
```

---

## 坑 2：`~/server/` 导入路径在 Nuxt 4 中解析失败 ⚠️ 教程错误

**现象**：服务端报 `Cannot find module '~/server/utils/prisma'`。

**原因**：Nuxt 4 中 `~/` 指向 `app/`，`~~/` 指向项目根。服务端文件在项目根 `server/` 下，不在 `app/` 内。

**修正**：所有 `~/server/` → `~~/server/`（涉及 7+ 个文件）。

---

## 坑 3：PrismaClient ESM 导入不兼容（CJS 模块冲突）

**现象**：`import { PrismaClient } from '@prisma/client'` 报错 `Named export 'PrismaClient' not found...is a CommonJS module`。

**原因**：pnpm 新版本解析后的 `@prisma/client` 是 CJS 模块，不支持 ESM named export。

**修正**：
```typescript
// server/utils/prisma.ts
import pkg from '@prisma/client'
const { PrismaClient } = pkg

const prisma = new PrismaClient()
export default prisma
```

---

## 坑 4（已消除）：Google Fonts / Google Icons 服务不可用

**注意**：降级到 `@nuxt/ui` v2 后，**此问题不再存在**。v2 不依赖 `@nuxt/fonts`，不会连接 Google CDN。无需任何额外配置。

如果你因某些原因必须使用 v3，需要双重配置：
```typescript
// nuxt.config.ts（仅 v3 需要）
icon: { serverBundle: 'local' },
fonts: { providers: { google: false, googleicons: false } },
```

---

## 坑 5（已消除）：Tailwind CSS 不生效——缺少 CSS 入口文件

**注意**：v2 的 `@nuxtjs/tailwindcss` 自动管理 Tailwind CSS，**无需手动创建 `main.css`**，也无需在 `nuxt.config.ts` 中配置 `css` 选项。

如果你因某些原因必须使用 v3：
```css
/* app/assets/css/main.css（仅 v3 需要） */
@import "tailwindcss";
@import "@nuxt/ui";
```

---

## 坑 6（已消除）：`<UFormGroup>` 组件不存在

**注意**：降级到 v2 后，`UFormGroup` **是正确且存在的组件**。之前报错是因为我们装了 v3（v3 中改名为 `UFormField`）。v2 完全支持教程原文的 `UFormGroup`。

---

## 坑 7（已消除）：`<UApp>` 包裹需求

**注意**：v2 不需要 `<UApp>` 根组件包裹。`app.vue` 直接用 `<NuxtLayout><NuxtPage /></NuxtLayout>` 即可。

---

## 坑 8（已消除）：Dashboard 表单无法交互

**现象（v3 特有问题）**：点击表单字段弹出灰色遮罩层，阻断所有输入。

**原因**：v3 的 `UModal` + `UFormField` 组合存在事件冲突——点击输入框会触发模态框的 overlay 事件。

**解决**：降级到 v2 后，模态框和表单工作完全正常。

原教程还有两个代码层面的写法可以优化（v2 下非强制，属于良好实践）：
1. `await useFetch` 阻塞渲染 → **建议**移除 `await`，用 `pending` 管理加载态（教程原文如此，v2 下实际也能工作，但去掉更规范）
2. `reactive()` → **建议**改用独立 `ref()`（在 teleported 组件中更可靠，教程原文用 `reactive`，v2 下不会崩溃但 `ref` 是更安全的写法）

---

## 坑 9：`pnpm install` 因代理配置连接失败

**现象**：`pnpm install` 报 `ECONNREFUSED 127.0.0.1:7897`。

**原因**：环境变量中配置了代理但服务未启动。修正：启动代理或移除 `HTTP_PROXY` 环境变量。

---

## 坑 10（非教程问题）：`pnpm approve-builds` 需要手动批准

**修正**：在 `package.json` 中添加：
```json
"pnpm": {
  "onlyBuiltDependencies": [
    "@prisma/client", "@prisma/engines", "prisma",
    "esbuild", "@parcel/watcher"
  ]
}
```

## 汇总：最低限度改动清单

如果从头搭建，只需要在教程的步骤上做以下改动：

| 序号 | 改动点 | 教程原文 | 实际应写 | 是否必须 |
|------|--------|----------|----------|----------|
| 1 | Nuxt UI 版本 | `pnpm add @nuxt/ui` | `pnpm add @nuxt/ui@^2.0.0` | **必须** ⚠️ 不锁版本会装 v3，整个项目无法运行 |
| 2 | Prisma 版本 | `pnpm add -D prisma`（不锁版本） | `pnpm add -D prisma@6.19.3` | **必须** ⚠️ Prisma 7 有破坏性变更 |
| 3 | Prisma 导入写法 | `import { PrismaClient } from '@prisma/client'` | `import pkg from '@prisma/client'`<br>`const { PrismaClient } = pkg` | **必须** ⚠️ pnpm 下 CJS 模块 named export 报错 |
| 4 | 服务端导入路径 | `~/server/utils/...` | `~~/server/utils/...` | **必须** ⚠️ Nuxt 4 中 `~/` 指向 `app/`，不是项目根 |
| 5 | SQLite 替换（可选） | PostgreSQL schema + 枚举 | SQLite provider + `status String` + `priority String`（**移除 enum 定义**） | 可选，选 SQLite 必须一并改 |
| 6 | Dashboard fetch 写法 | `await useFetch` + `reactive()` | `useFetch`（无 await）+ 独立 `ref()` | **建议**（v2 下不崩溃，但去掉 await 更规范；`ref()` 在 teleported modal 中更可靠）|

> **第 5 条补充说明**：SQLite 不支持原生枚举。如果选择 SQLite，除了改 `provider` 和 `DATABASE_URL`，还必须将 `schema.prisma` 中的 `status TaskStatus` 改为 `status String @default("TODO")`、`priority Priority` 改为 `priority String @default("MEDIUM")`，并删除 `enum TaskStatus` 和 `enum Priority` 的定义块。详见 `DATABASE_SWITCH.md`。
